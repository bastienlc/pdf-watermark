# PDF 浮水印工具 GUI 設計文件

**日期：** 2026-05-19
**狀態：** 已核准，待實作

---

## 概述

為 `pdf-watermark` CLI 工具建立本地網頁 GUI，讓不熟悉命令列的使用者可以透過瀏覽器操作，重點支援繁體中文浮水印。

---

## 技術選型

| 項目 | 選擇 | 理由 |
|------|------|------|
| 後端框架 | Flask | 輕量、Python 原生、單一新增依賴 |
| 前端響應 | Alpine.js（本地端） | 15KB、無 build 步驟、折疊/響應狀態簡潔 |
| PDF 預覽 | PDF.js（本地端） | 離線可用、繁中字體渲染完美、無需 Poppler |
| 新增依賴 | `flask` only | 不影響現有 CLI 功能 |

Alpine.js 和 PDF.js 皆下載至 `static/` 目錄，**完全離線可用**。

## API 路由

| 方法 | 路徑 | 說明 |
|------|------|------|
| `GET` | `/` | 回傳主頁面 HTML |
| `POST` | `/api/upload` | 接收拖曳上傳的 PDF，存至 temp dir，回傳 `{ temp_path }` |
| `POST` | `/api/preview` | 以目前參數產生暫存浮水印 PDF，回傳 PDF bytes |
| `POST` | `/api/process` | 正式輸出浮水印 PDF 至指定路徑，回傳 `{ output_path, count }` |

所有路由直接呼叫現有的 `add_watermark_from_options()`，不重複邏輯。

## 核心互動行為

### 即時預覽
- 任一參數變動 → Alpine.js debounce **500ms** → `POST /api/preview`
- 右側上方顯示**原始 PDF**，下方顯示**加水印後**，方便對比
- 預覽失敗時右側顯示錯誤訊息，不中斷操作

### 檔案輸入（雙模式）
- **拖曳上傳**：支援單一 PDF 檔案。Flask 存至 `tempfile.mkdtemp()`，回傳 temp_path，後續用此路徑操作；暫存檔於 Flask app 關閉時清除
- **路徑輸入**：支援單一 PDF 檔案或整個資料夾（批次處理）。驗證路徑存在後直接讀取，不複製檔案
- 兩者互相連動：拖曳後自動填入路徑欄；資料夾路徑無法拖曳，必須使用路徑輸入

### 進階選項（預設折疊）
- 格狀模式：水平/垂直重複數量、是否留邊距
- 定位模式：x、y 座標（0~1）、水平對齊
- 平行處理：`--workers` 數量
- 文字不可選取：`--unselectable`

## 繁體中文三層保障

| 層級 | 實作 |
|------|------|
| **UI 介面** | `<html lang="zh-Hant">` + UTF-8，全介面繁中標籤 |
| **字體選單** | CJK 字體（STSong-Light、MSung-Light 等）置頂並標示「✓ 支援中文」；Helvetica 等標示「僅支援英文」 |
| **浮水印邏輯** | 沿用已修復的 `contains_cjk()` 自動偵測，偵測到中文且使用英文字體時，UI 顯示黃色提示訊息 |

---

## 錯誤處理

| 情況 | 處理方式 |
|------|---------|
| 路徑不存在 | 輸入框紅框 + 即時文字提示 |
| 無效顏色值 | 恢復上一個有效值 |
| 預覽產生失敗 | 右側顯示錯誤訊息，不中斷操作 |
| 批次處理進度 | 顯示「第 X / 共 Y 個檔案」進度條 |
| 輸出路徑無寫入權限 | process API 回傳錯誤，UI 顯示說明 |

## 啟動方式

```powershell
# 安裝後
uv run watermark-gui
# 自動開啟 http://localhost:7860
```

預設 port 7860（避免與常見 8080 衝突），可透過環境變數 `WATERMARK_GUI_PORT` 覆蓋。啟動後自動呼叫 `webbrowser.open()` 開啟瀏覽器。
